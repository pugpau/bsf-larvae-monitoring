## 1. プロジェクト概要

### 1.1 背景と課題

菌体リン酸肥料および軽量盛土材の製造において、以下の課題が存在します：

#### 現状の課題
1. **配合の属人化**
   - 熟練者の勘と経験に依存した配合決定
   - 若手への技術継承が困難
   - 担当者不在時の製造停滞リスク

2. **原料成分の変動対応**
   - 原料ごとに成分値が変動
   - 都度の試行錯誤による調整
   - 品質の安定化が困難

3. **製造履歴管理の不備**
   - 紙ベースまたはExcel管理
   - ロットごとのトレーサビリティ不足
   - 品質クレーム発生時の原因究明が困難

4. **品質保証体制の弱さ**
   - 出荷時の成分証明書作成に時間を要する
   - データの一元管理ができていない
   - 過去データの活用が不十分

### 1.2 目的

本プロジェクトは、AIによる配合最適化と製造データの一元管理により、以下を実現することを目的とします：

**目指す姿**：
1. **配合の最適化と標準化**
   - AIが原料成分の変動を学習し、最適配合を自動提案
   - 熟練者の勘に依存しない再現性の高い製造
   - 若手主体でも高品質な製造が可能

2. **完全なトレーサビリティ**
   - 原料入荷→配合→製造→出荷までの全工程を記録
   - ロット単位での製造履歴・成分データの即時検索
   - 品質クレーム時の迅速な原因究明

3. **品質保証体制の強化**
   - 出荷時の成分証明書を自動生成
   - 品質データの蓄積と傾向分析
   - 品質クレームの予防

4. **業務効率化**
   - 配合計算の自動化（手作業削減）
   - データ入力作業の効率化
   - レポート作成の自動化

### 1.3 期待される効果

| 効果項目 | 現状 | 導入後 | 改善効果 |
|---------|------|--------|---------|
| 配合決定時間 | 30分/ロット | 5分/ロット | **80%削減** |
| 品質クレーム | 年間10件 | 年間3件以下 | **70%削減** |
| 成分証明書作成 | 30分/件 | 5分/件 | **80%削減** |
| 若手育成期間 | 3年 | 6ヶ月 | **75%短縮** |
| データ検索時間 | 15分/件 | 1分/件 | **90%削減** |

**年間経済効果（試算）**: 約500万円
- 作業時間削減: 200万円
- 品質クレーム削減: 150万円
- 原料ロス削減: 100万円
- その他効率化: 50万円

#### 2.2.3 データベース設計

**データベース**: PostgreSQL 14+

**主要テーブル**:
1. **原料マスタ** (materials)
   - 原料ID, 原料名, 標準成分値（N, P, K等）, 単価, 供給元

2. **原料入荷** (material_receipts)
   - 入荷ID, 原料ID, ロット番号, 入荷日, 数量, 成分分析値

3. **在庫** (inventory)
   - 在庫ID, 原料ID, ロット番号, 在庫数量, 更新日時

4. **配合レシピ** (blend_recipes)
   - レシピID, 製品種類, レシピ名, 目標成分値, 作成日

5. **配合明細** (blend_recipe_details)
   - 明細ID, レシピID, 原料ID, 配合比率

6. **製造ロット** (production_lots)
   - ロットID, 製品種類, 製造日, 製造量, 担当者, ステータス

7. **製造実績** (production_records)
   - 実績ID, ロットID, 原料ID, 使用ロット番号, 使用量, 配合比率

8. **品質データ** (quality_data)
   - 品質ID, ロットID, 分析日, 成分分析値, 品質検査結果

9. **出荷** (shipments)
   - 出荷ID, ロットID, 出荷日, 出荷先, 出荷量, 証明書番号

10. **ユーザー** (users)
    - ユーザーID, ユーザー名, メール, パスワード, ロール

11. **操作ログ** (audit_logs)
    - ログID, ユーザーID, 操作種類, 操作日時, 操作内容

**データ保持期間**: 10年間
