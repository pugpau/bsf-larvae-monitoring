## 1. プロジェクト概要

### 1.1 背景
ブラックソルジャーフライ（BSF）幼虫の養殖は、食品廃棄物のリサイクルと高タンパク飼料生産を両立する循環型ビジネスとして注目されています。しかし、養殖環境の最適化には温度・湿度・気圧・ガス濃度（H2S、NH3）などの複数パラメータを常時監視し、基質（飼料）配合との相関を分析する必要があります。

現状では手作業による定期測定に依存しており、以下の課題があります：
- リアルタイムでの環境変化の把握が困難
- データの蓄積と分析が体系的に行われていない
- 養殖効率（成長速度、生存率）の最適化が困難
- 異常検知の遅れによる損失リスク

### 1.2 目的
本プロジェクトは、IoT技術を活用してBSF養殖環境を24時間365日監視し、データに基づく養殖最適化を実現することを目的とします。

**期待される効果**：
- 養殖効率の向上：成長速度15-20%改善
- 異常検知による損失防止：年間損失額30%削減
- 作業時間削減：環境測定作業の90%自動化
- データドリブンな意思決定：基質配合の最適化

## 2. システム要件

### 2.1 ハードウェア要件

#### 2.1.1 IoTセンサーデバイス（M5StickC + BME680）
- **数量**: 10台（初期導入）、将来的に50台まで拡張可能
- **仕様**:
  - デバイス: M5StickC（ESP32ベース）
  - センサー: BME680（温度・湿度・気圧・ガスセンサー）
  - 通信: WiFi（2.4GHz）+ MQTT over TLS/SSL
  - 測定間隔: 2秒（平均化30秒ごとに送信）
  - 電源: USB給電またはバッテリー（4-6時間）
  - 3D位置情報: X, Y, Z座標による配置管理

#### 2.1.2 サーバー・インフラ
- **アプリケーションサーバー**:
  - CPU: 4コア以上
  - メモリ: 16GB以上
  - ストレージ: SSD 500GB以上
- **データベースサーバー**:
  - PostgreSQL: リレーショナルデータ管理
  - InfluxDB: 時系列データ管理
- **MQTTブローカー**:
  - Mosquitto（TLS/SSL対応）
  - 同時接続: 100デバイス以上対応

#### 2.1.3 ネットワーク機器
- 無線LANアクセスポイント: 2.4GHz対応、養殖施設全域カバー
- ネットワークスイッチ: Gigabit Ethernet
- ルーター: VPN対応

### 2.2 ソフトウェア要件

#### 2.2.1 バックエンド（APIサーバー）
- **言語・フレームワーク**: Python 3.10+ / FastAPI
- **主要機能**:
  - REST API提供（センサーデータCRUD、基質管理、デバイス管理）
  - MQTT通信（QoS 1、永続セッション、LWT対応）
  - 認証・認可（JWT、RBAC）
  - データベース管理（PostgreSQL + InfluxDB）
  - リアルタイムデータ処理
  - アラート機能（閾値監視、通知）
  - データエクスポート（CSV、JSON）

#### 2.2.2 フロントエンド（Webアプリケーション）
- **言語・フレームワーク**: React 18+ / Material-UI
- **主要機能**:
  - リアルタイムダッシュボード
    - 温度・湿度・気圧・ガス濃度の可視化
    - 時系列グラフ（Chart.js / Plotly.js）
    - 3D可視化（デバイス配置）
  - 基質管理機能
    - 基質タイプ管理（原材料、栄養価）
    - バッチ管理（投入日時、配合比率）
    - 基質とセンサーデータの相関分析
  - デバイス管理機能
    - デバイス一覧・詳細表示
    - ステータス監視（オンライン/オフライン）
    - 3D位置情報設定
  - アラート設定・履歴表示
  - ユーザー管理（管理者/オペレーター権限）
  - レスポンシブデザイン（PC/タブレット対応）

#### 2.2.3 IoTデバイスファームウェア
- **言語・環境**: Arduino IDE / PlatformIO
- **主要機能**:
  - WiFi自動接続・再接続（指数バックオフ）
  - NTP時刻同期
  - BME680センサー制御
    - 温度・湿度・気圧測定
    - H2S・NH3ガス濃度測定
    - ゼロキャリブレーション
  - MQTT通信（QoS 1、永続セッション、LWT）
  - オフライン時データ保持（リングバッファ10件）
  - 液晶ディスプレイ表示
    - センサー値表示
    - MQTT接続状態表示
    - バッファ状態表示

#### 2.2.4 データベース設計
- **PostgreSQL（リレーショナルDB）**:
  - ユーザー管理テーブル
  - デバイス管理テーブル（3D位置情報、ステータス）
  - 基質タイプ・バッチ管理テーブル
  - アラート設定・履歴テーブル
- **InfluxDB（時系列DB）**:
  - センサー測定値（温度、湿度、気圧、ガス濃度）
  - 高速クエリ・集計機能
  - データ保持期間: 2年間

### 2.3 インフラ・運用要件

#### 2.3.1 デプロイメント環境
- **Docker環境**: docker-compose による一括管理
- **コンテナ構成**:
  - FastAPI バックエンド
  - React フロントエンド（Nginx）
  - PostgreSQL
  - InfluxDB
  - MQTT Broker（Mosquitto）

#### 2.3.2 セキュリティ
- MQTT通信の暗号化（TLS/SSL）
- API通信の暗号化（HTTPS）
- JWT認証によるアクセス制御
- ロールベースアクセス制御（RBAC）
- MQTT ACL（デバイスごとのトピック制限）

#### 2.3.3 バックアップ・復旧
- PostgreSQLデータベース: 日次自動バックアップ
- InfluxDBデータベース: 週次自動バックアップ
- バックアップ保持期間: 3ヶ月
- 復旧手順書の提供
