## 1. 認証 (Authentication)

### AT-AUTH-001: ログイン成功
- **重要度**: A (必須)
- **前提条件**: ユーザー demo/demo が登録済み
- **手順**:
  1. ログインページ (`/login`) にアクセス
  2. ユーザー名に「demo」を入力
  3. パスワードに「demo」を入力
  4. 「ログイン」ボタンをクリック
- **期待結果**:
  - ダッシュボード画面に遷移する
  - 「ERC製品管理システム」ヘッダーが表示される
  - 5つのタブ（搬入予定、配合管理、分析ダッシュボード、品質管理、マスタ管理）が表示される
  - 「搬入予定」タブがデフォルトで選択状態になっている

### AT-AUTH-002: ログイン失敗（パスワード不正）
- **重要度**: A (必須)
- **前提条件**: ユーザー demo が登録済み
- **手順**:
  1. ログインページにアクセス
  2. ユーザー名「demo」、パスワード「wrong」を入力
  3. 「ログイン」ボタンをクリック
- **期待結果**:
  - エラーメッセージが表示される
  - ログインページに留まる（ダッシュボードには遷移しない）
  - パスワード入力欄がクリアされる

### AT-AUTH-003: トークンリフレッシュ
- **重要度**: B (重要)
- **前提条件**: ログイン済み状態
- **手順**:
  1. ログイン後、アクセストークンの有効期限（30分）まで待機
  2. いずれかのAPI操作（タブ遷移、データ取得等）を実行
- **期待結果**:
  - httpOnly cookie のリフレッシュトークンにより自動的に新しいアクセストークンが取得される
  - 操作が正常に完了する（再ログインは不要）

### AT-AUTH-004: ログアウト
- **重要度**: A (必須)
- **前提条件**: ログイン済み状態
- **手順**:
  1. 右上のユーザーメニューボタンをクリック
  2. 「ログアウト」メニュー項目をクリック
- **期待結果**:
  - ログインページに遷移する
  - ブラウザを更新してもダッシュボードにアクセスできない
  - セッション情報がクリアされる

### AT-AUTH-005: 未認証アクセス拒否
- **重要度**: A (必須)
- **前提条件**: 未ログイン状態
- **手順**:
  1. ブラウザで直接ダッシュボードURLにアクセス
- **期待結果**:
  - ログインページにリダイレクトされる
  - API呼び出しは401エラーを返す

## 2. Tab 0: 搬入管理

### AT-DEL-001: 搬入予定一覧表示
- **重要度**: A (必須)
- **前提条件**: ログイン済み、搬入予定データが存在
- **手順**:
  1. ログイン後、「搬入予定」タブ（デフォルト）を確認
- **期待結果**:
  - 搬入予定一覧がテーブル形式で表示される
  - 各行に搬入先名、搬入物名、予定日、推定重量、ステータスが表示される
  - ページネーション（表示件数切替: 10/25/50/100件）が機能する
  - テーブル/週間/月間のビュー切替ボタンが表示される

### AT-DEL-002: 搬入予定新規登録
- **重要度**: A (必須)
- **前提条件**: ログイン済み、搬入先マスターと搬入物マスターにデータ登録済み
- **手順**:
  1. 「新規登録」ボタンをクリック
  2. 搬入先を選択（カスケード選択）
  3. 搬入物カテゴリと搬入物名を選択
  4. 予定日を入力
  5. 推定重量を入力
  6. 「保存」ボタンをクリック
- **期待結果**:
  - 搬入予定登録フォームが表示される
  - 搬入先選択後、対応する搬入物カテゴリと搬入物名がカスケードで絞り込まれる
  - 保存後、一覧に新しいレコードが表示される
  - ステータスは「予定」(scheduled) で作成される

### AT-DEL-003: 搬入予定検索
- **重要度**: B (重要)
- **前提条件**: ログイン済み、搬入予定データが5件以上存在
- **手順**:
  1. 検索ボックスに搬入先名の一部を入力
  2. 検索結果を確認
- **期待結果**:
  - 入力した文字列に一致する搬入予定のみが一覧に表示される
  - 検索は部分一致（ILIKE）で動作する

### AT-DEL-004: 搬入予定CSV出力
- **重要度**: B (重要)
- **前提条件**: ログイン済み、搬入予定データが存在
- **手順**:
  1. 「CSV出力」ボタンをクリック
- **期待結果**:
  - BOM付きUTF-8のCSVファイルがダウンロードされる
  - ファイルにはID、搬入先名、搬入物カテゴリ、搬入物名、予定日、推定重量、実重量、ステータス等のカラムが含まれる
  - 日本語が文字化けしない（Excel で正常に開ける）

### AT-DEL-005: ステータス更新 (scheduled→delivered)
- **重要度**: A (必須)
- **前提条件**: ログイン済み、ステータスが「予定」の搬入予定が存在
- **手順**:
  1. 搬入予定一覧で対象レコードのステータス変更ボタンをクリック
  2. 新しいステータス「搬入済」を選択
  3. 実重量を入力して確定
- **期待結果**:
  - ステータスが「搬入済」(delivered) に更新される
  - 対応するWasteRecordが自動作成される（status=pending）
  - 一覧のステータス表示が更新される

### AT-DEL-006: 配合開始ボタン→Tab1連携
- **重要度**: A (必須)
- **前提条件**: ログイン済み、ステータスが「配合準備完了」(ready_for_formulation) の搬入予定が存在
- **手順**:
  1. 対象レコードの「配合開始」ボタンをクリック
- **期待結果**:
  - Tab 1（配合管理）に自動遷移する
  - AI推薦ダイアログが対象WasteRecordを選択した状態で自動表示される
  - 「推薦を実行」ボタンが有効になっている

## 3. Tab 1: 配合管理

### AT-FORM-001: AI推薦ダイアログ表示
- **重要度**: A (必須)
- **前提条件**: ログイン済み、分析済みWasteRecordが存在
- **手順**:
  1. 配合管理タブに遷移
  2. 「AI推薦」ボタンをクリック
- **期待結果**:
  - 配合推薦ダイアログが表示される
  - 搬入記録のセレクトボックスと候補数の入力欄が表示される
  - 搬入記録を選択するまで「推薦を実行」ボタンは無効（disabled）

### AT-FORM-002: AI推薦の実行と候補表示
- **重要度**: A (必須)
- **前提条件**: ログイン済み、分析済みWasteRecordが3件以上存在
- **手順**:
  1. AI推薦ダイアログを開く
  2. 搬入記録を選択
  3. 「推薦を実行」ボタンをクリック
- **期待結果**:
  - ML予測、類似度分析、ルールベース、最適化の4種類の推薦候補が表示される
  - 各候補に固化材種類、添加量、推定コスト、信頼度が表示される
  - 候補ごとに「採用」ボタンが表示される

### AT-FORM-003: 推薦候補の承認 (Accept)
- **重要度**: A (必須)
- **前提条件**: AI推薦結果が表示された状態
- **手順**:
  1. 推薦候補の1つの「採用」ボタンをクリック
- **期待結果**:
  - 配合レコードが「提案」(proposed) ステータスで作成される
  - 一覧に新しいレコードが表示される
  - その後「承認」操作でステータスが「承認済」(accepted) に遷移する

### AT-FORM-004: 配合の却下 (Reject)
- **重要度**: B (重要)
- **前提条件**: ステータスが「提案」の配合レコードが存在
- **手順**:
  1. 対象レコードの「却下」ボタンをクリック
  2. 却下理由を入力
  3. 確定
- **期待結果**:
  - ステータスが「却下」(rejected) に更新される
  - 却下理由がnotesに記録される
  - 一覧のステータス表示が更新される

### AT-FORM-005: 配合の適用 (Apply)
- **重要度**: A (必須)
- **前提条件**: ステータスが「承認済」の配合レコードが存在
- **手順**:
  1. 対象レコードの「適用」ボタンをクリック
  2. 実際の配合内容（固化材種類、添加量）を入力
  3. 実コストを入力
  4. 確定
- **期待結果**:
  - ステータスが「適用済」(applied) に更新される
  - actual_formulation と actual_cost が記録される
  - 対応するWasteRecordのステータスが「配合済」(formulated) に更新される

### AT-FORM-006: 溶出検証 (Verify)
- **重要度**: A (必須)
- **前提条件**: ステータスが「適用済」の配合レコードが存在
- **手順**:
  1. 対象レコードの「検証」ボタンをクリック
  2. 溶出試験結果（Pb, As, Cd, Cr6, Hg, Se, F, B）を入力
  3. 合否を選択して確定
- **期待結果**:
  - ステータスが「検証済」(verified) に更新される
  - elution_result と elution_passed が記録される
  - 基準値超過がある場合は超過項目が警告表示される

### AT-FORM-007: 配合詳細ドロワー
- **重要度**: B (重要)
- **前提条件**: 配合レコードが存在
- **手順**:
  1. 配合一覧で任意のレコードの行をクリック
- **期待結果**:
  - 詳細ドロワーが右側からスライドインで表示される
  - Stepperタイムラインでステータス遷移履歴（proposed→accepted→applied→verified）が表示される
  - 検証済みの場合、溶出結果テーブルが表示される
  - コスト情報（推定コスト、実コスト）が表示される

### AT-FORM-008: ステータスフィルタ
- **重要度**: B (重要)
- **前提条件**: 複数ステータスの配合レコードが存在
- **手順**:
  1. 「ステータス」フィルタで「承認済」を選択
  2. 表示結果を確認
  3. 「全て」に戻す
- **期待結果**:
  - フィルタ選択に応じた配合レコードのみが一覧に表示される
  - 「全て」を選択すると全レコードが表示される
  - フィルタ選択肢: 全て、提案、承認済、適用済、検証済、却下

  3. 配合詳細（固化材/抑制剤とその添加率）を設定
- **期待結果**:
  - レシピ一覧が表示される
  - 新規レシピが作成できる
  - 配合詳細（固化材/抑制剤とその添加率）が設定できる
  - バージョン管理が適用される（更新時に新バージョンが作成される）

## 8. エラーハンドリング

### AT-ERR-001: バリデーションエラー表示
- **重要度**: A (必須)
- **前提条件**: ログイン済み
- **手順**:
  1. 搬入予定の新規登録フォームを開く
  2. 必須項目を空のまま「保存」をクリック
- **期待結果**:
  - フォームにバリデーションエラーメッセージが表示される
  - 未入力の必須項目が赤枠等で強調表示される
  - 保存処理は実行されない（APIリクエストが送信されない）

### AT-ERR-002: API接続エラー表示
- **重要度**: B (重要)
- **前提条件**: ログイン済み、バックエンドが停止中
- **手順**:
  1. バックエンドサーバーを停止
  2. いずれかのタブでデータ取得操作を実行
- **期待結果**:
  - エラーメッセージ（「サーバーに接続できません」等）が表示される
  - アプリケーション全体がクラッシュしない
  - ErrorBoundaryによるフォールバック表示が機能する

### AT-ERR-003: ErrorBoundary動作確認
- **重要度**: A (必須)
- **前提条件**: ログイン済み
- **手順**:
  1. 全5タブを順番に遷移
  2. 各タブでスクロール操作を実行
- **期待結果**:
  - いずれのタブでも「エラーが発生しました」というErrorBoundary画面が表示されない
  - 正常なコンテンツまたはEmptyState/ローディングスケルトンが表示される
