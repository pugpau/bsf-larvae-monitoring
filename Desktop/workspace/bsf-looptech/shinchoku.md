# 開発の進捗

## タスクの概要

基質管理システムのフロントエンド開発

## 進捗

1.  **初期設定:**
    *   プロジェクトのファイル構造を理解
        *   `frontend/`ディレクトリ内のファイル構成を把握
        *   `package.json`の内容を確認し、依存関係を把握
    *   必要な依存関係を確認
        *   Axios, Material-UIなどのライブラリがインストールされていることを確認
2.  **基本的なUIの構築:**
    *   App.js, index.jsの作成
        *   `index.js`: Reactアプリケーションのエントリーポイントとして、`App.js`をレンダリングする
        *   `App.js`:  Material-UIの`Container`, `AppBar`, `Toolbar`, `Typography`, `Tabs`, `Tab`, `Box`コンポーネントを使用して基本的なレイアウトを構築
        *   タブ切り替え機能を実装 (useStateによる状態管理)
    *   Material-UIの導入
        *   `@mui/material`と`@emotion/react`などのMUI関連パッケージをインストール
        *   `createTheme`を使用してテーマを設定し、アプリケーション全体に適用
        *   `CssBaseline`を使用して、ブラウザのデフォルトスタイルをリセット
    *   基質タイプと基質バッチ登録フォームの作成
        *   各フォームの基本的なレイアウトと`TextField`, `Select`, `Button`などのMUIコンポーネントの配置
        *   フォームの入力項目の状態管理 (useState)
        *   `AddIcon`と`DeleteIcon`のインポートと使用
3.  **詳細UIの構築:**
    *   SubstrateTypeForm.jsとSubstrateBatchForm.jsの作成
        *   **SubstrateTypeForm.js**:
            *   基質タイプ名の入力
            *   基質カテゴリの選択（Selectコンポーネント、`MenuItem`）
            *   説明文の入力 (TextField, multiline)
            *   属性の動的追加機能 (属性名、値、単位)
                *   属性の追加、削除、更新機能を実装 (useState, IconButton, DeleteIcon, AddIcon)
        *   **SubstrateBatchForm.js**:
            *   ファームIDの入力
            *   バッチ名と説明の入力
            *   総重量と単位の設定 (Selectコンポーネント、`MenuItem`)
            *   バッチ番号と保管場所の入力
            *   基質コンポーネントの動的追加機能 (基質タイプ、比率)
                *   基質タイプの選択 (APIから取得した基質タイプ一覧をSelectコンポーネントで表示)
                *   比率の合計が100%になるバリデーション
            *   useEffectフックを用いてAPIから基質タイプの一覧を取得
4.  **API連携の実装:**
    *   SubstrateTypeForm.jsとSubstrateBatchForm.jsにAxiosによるAPIリクエストを実装
        *   axios.post()を使用してAPIにデータを送信
        *   登録成功/失敗時のメッセージ表示
        *   Snackbarコンポーネントを使用して成功/エラーメッセージを表示
        *   編集機能の実装
    *   APIエンドポイント: `/api/substrate/types`, `/api/substrate/batches`
5.  **一覧表示機能の実装:**
    *   SubstrateTypeList.jsとSubstrateBatchList.jsの作成
        *   基質タイプと基質バッチの一覧表示
            *   Table, TableHead, TableBody, TableRow, TableCellコンポーネントを使用してテーブル形式で表示
            *   EditIconとDeleteIconのインポートと使用
        *   削除機能の実装（確認ダイアログ付き）
            *   useStateを使用して削除対象のIDを保持
            *   Dialogコンポーネントを使用して確認ダイアログを表示
            *   axios.delete()を使用してAPIに削除リクエストを送信
        *   詳細表示（基質バッチのみ、ダイアログ表示）
            *   useStateを使用して詳細情報を表示
            *   Dialogコンポーネントを使用して詳細情報を表示
        *   useEffectフックを使用したAPIからのデータ取得
            *   axios.get()を使用してAPIからデータを取得し、useStateで状態を管理
        *   編集機能の実装
            *   SubstrateTypeForm.jsとSubstrateBatchForm.jsを編集モードに対応させ、編集対象のデータを初期値として設定
6.  **問題解決:**
    *   依存関係の問題を解決するためにnpm install、react-scriptsの再インストールとバージョンの調整を行った。
    *   開発サーバー起動時のエラーを修正: 3000番ポートの競合を解決するために、pkillでプロセスを停止し、react-scriptsを再起動した。
