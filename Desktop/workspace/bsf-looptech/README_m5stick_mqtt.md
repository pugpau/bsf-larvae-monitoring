# M5StickC BSF-LoopTech MQTT センサー接続ガイド

このドキュメントでは、M5StickCとBME680/BME688ガスセンサーを使用して、BSF-LoopTechバックエンドシステムにMQTT経由でデータを送信する方法について説明します。

## 概要

`m5stick_bsf_mqtt.ino`スケッチは、M5StickCデバイスがBME680/BME688センサーからの測定データを収集し、MQTTプロトコルを使用してBSF-LoopTechバックエンドに送信するためのものです。このスケッチは、以前のGoogle Sheetsへのデータ送信から、BSF-LoopTechのMQTTベースのセンサーデータ管理システムへの移行を実現します。

## 必要なハードウェア

- M5StickC
- BME680/BME688ガスセンサー
- Wi-Fi接続

## 必要なライブラリ

以下のライブラリをArduino IDEにインストールする必要があります：

- M5StickC
- Wire
- EEPROM
- WiFi
- PubSubClient (MQTT用)
- ArduinoJson
- Adafruit_BME680

## 設定

スケッチを使用する前に、以下の設定を行う必要があります：

### デバイス設定

```cpp
const char* DEVICE_ID = "taka001";  // デバイスID
const char* DEVICE_TYPE = "gas_sensor";  // デバイスタイプ
const char* FARM_ID = "farm123";  // ファームID
```

- `DEVICE_ID`: システム内でのデバイスの一意の識別子
- `DEVICE_TYPE`: デバイスのタイプ（gas_sensor, temperature_sensor など）
- `FARM_ID`: デバイスが属する農場/施設のID

### Wi-Fi設定

```cpp
const char* ssid = "TP-LINK_6A98";  // Wi-Fi SSID
const char* password = "kohakukohaku";  // Wi-Fiパスワード
```

お使いのWi-Fiネットワークの認証情報に変更してください。

### MQTT設定

```cpp
const char* mqtt_server = "bsf-server";  // MQTTブローカーのホスト名
const int mqtt_port = 1883;  // MQTTブローカーポート
const char* mqtt_user = NULL;  // MQTTユーザー名（必要な場合）
const char* mqtt_password = NULL;  // MQTTパスワード（必要な場合）
```

- `mqtt_server`: BSF-LoopTechバックエンドが実行されているサーバーのホスト名です。デフォルトでは`bsf-server`に設定されていますが、実際の環境に合わせて変更してください。
- `mqtt_port`: MQTTブローカーのポート（通常は1883）
- `mqtt_user`と`mqtt_password`: MQTTブローカーが認証を必要とする場合は設定してください

### ホスト名の設定

MQTTブローカーを実行するサーバー側では、以下の設定が必要です：

1. サーバーのホスト名を`bsf-server`に設定するか、スケッチ内の`mqtt_server`変数を実際のホスト名に合わせて変更してください。

2. ホスト名解決のための設定
   - ローカルネットワーク内のDNSサーバーにホスト名を登録する
   - または、M5StickCのWi-Fi設定で固定IPアドレスを使用する

## 機能

### センサー測定

このスケッチは以下のパラメータを測定します：

- 温度（°C）
- 湿度（%）
- 気圧（hPa）
- ガス抵抗値（ohm）
- H2S濃度（ppm）- 計算値
- NH3濃度（ppm）- 計算値

### 動作モード

デバイスには以下の動作モードがあります：

1. **モード選択**: 起動時のデフォルトモード。H2SまたはNH3測定モードを選択できます。
2. **H2S測定**: 硫化水素（H2S）の測定に最適化されたヒータープロファイルを使用します。
3. **NH3測定**: アンモニア（NH3）の測定に最適化されたヒータープロファイルを使用します。

### ゼロキャリブレーション

正確な測定のために、きれいな空気環境でゼロキャリブレーションを実行できます。キャリブレーションデータはEEPROMに保存され、デバイスの再起動後も保持されます。

### データ送信

測定データは30秒ごとに平均化され、以下の形式でMQTTトピックに送信されます：

```
bsf/{farm_id}/{device_type}/{device_id}
```

例：`bsf/farm123/gas_sensor/taka001`

データはJSON形式で送信され、以下のような構造になります：

```json
{
  "readings": [
    {
      "type": "temperature",
      "value": 25.5,
      "unit": "°C"
    },
    {
      "type": "humidity",
      "value": 65.2,
      "unit": "%"
    },
    {
      "type": "pressure",
      "value": 1013.4,
      "unit": "hPa"
    },
    {
      "type": "h2s_resistance",
      "value": 12345.6,
      "unit": "ohm"
    },
    {
      "type": "h2s",
      "value": 0.123,
      "unit": "ppm"
    }
  ],
  "timestamp": "2025-04-03T12:34:56Z"
}
```

## 使用方法

### ボタン操作

M5StickCには2つのボタンがあります：

- **ボタンA（メインボタン）**:
  - モード選択画面で押すと、H2S測定モードに入ります
  - 測定モード中に押すと、ゼロキャリブレーションを実行します

- **ボタンB（サイドボタン）**:
  - モード選択画面で押すと、H2SとNH3モードを切り替えます
  - 測定モード中に押すと、H2SとNH3測定モードを切り替えます

### ディスプレイ情報

測定モード中、ディスプレイには以下の情報が表示されます：

- 現在時刻
- 測定モード（H2SまたはNH3）
- デバイスID
- 温度
- 湿度
- ガス濃度（H2SまたはNH3）

## トラブルシューティング

### Wi-Fi接続の問題

デバイスがWi-Fiに接続できない場合：
- SSIDとパスワードが正しいことを確認してください
- M5StickCがWi-Fiルーターの範囲内にあることを確認してください
- Wi-Fiネットワークが2.4GHz帯域を使用していることを確認してください（M5StickCは5GHzをサポートしていません）

### MQTT接続の問題

MQTTブローカーに接続できない場合：
- MQTTサーバーのホスト名が正しいことを確認してください
- ホスト名がネットワーク内で正しく解決されることを確認してください
- MQTTブローカーが実行中であることを確認してください
- ファイアウォールがMQTTポート（通常は1883）をブロックしていないことを確認してください
- 認証が必要な場合は、ユーザー名とパスワードが正しいことを確認してください

### ホスト名解決の問題

ホスト名が解決できない場合：
- ローカルネットワークのDNSサーバーにホスト名が登録されていることを確認してください
- `/etc/hosts`ファイル（LinuxやmacOS）または`C:\Windows\System32\drivers\etc\hosts`ファイル（Windows）にホスト名とIPアドレスのマッピングを追加することを検討してください
- 代替手段として、スケッチ内の`mqtt_server`変数をIPアドレスに変更することも可能です（例：`"192.168.1.100"`）

### センサーの問題

BME680/BME688センサーが検出されない場合：
- センサーの配線が正しいことを確認してください（I2Cピン：SDA=32, SCL=33）
- センサーの電源が正しく供給されていることを確認してください
- I2Cアドレスが正しいことを確認してください（デフォルトは0x76または0x77）

## BSF-LoopTechバックエンドとの統合

このスケッチは、BSF-LoopTechバックエンドのMQTTメッセージ処理機能と互換性があります。バックエンドは、受信したMQTTメッセージを処理し、センサーデータをデータベースに保存します。

バックエンドでは、以下の処理が行われます：

1. MQTTトピックからfarm_id、device_type、device_idを抽出
2. JSONペイロードからセンサー測定値を抽出
3. 各測定値をデータベースに保存
4. 必要に応じてアラートを生成（閾値を超えた場合）

## 注意事項

- このスケッチを使用する前に、BSF-LoopTechバックエンドシステムにセンサーデバイスを登録する必要があります
- MQTTブローカーのホスト名は、実際の展開環境に合わせて変更する必要があります
- Wi-Fi認証情報は、使用するネットワークに合わせて変更する必要があります
- ガス濃度の計算は近似値であり、正確な測定のためには定期的なキャリブレーションが必要です
