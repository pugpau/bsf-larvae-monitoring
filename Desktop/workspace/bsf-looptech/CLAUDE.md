# ポート使用確認ルール

## 目的
開発環境やサーバーを起動する前に、使用予定のポートが他のアプリケーションで使用されていないことを確認する

## 実装方法
1. **確認対象**: 
   - フロントエンドポート（3000, 3001, 5173など）
   - バックエンドAPIポート（8000, 8001, 8080など）
   - その他のサービスポート
2. **確認コマンド**:
   ```bash
   lsof -i :ポート番号 | grep LISTEN
   ```
3. **使用中の場合の対応**:
   - 代替ポートへの変更を提案
   - docker-compose.ymlやその他の設定ファイルを適切に更新
   - 環境変数の更新（VITE_API_BASE_URLなど）

## 運用ルール
- ポートを使用するサービスを起動する前に必ず確認
- 使用中のポートを発見した場合は、ユーザーに通知してから変更
- 変更後は、関連する全ての設定を更新

# 会話記録ルール

## 目的
プロジェクトの進捗、決定事項、技術的な議論を継続的に記録し、後から参照可能な状態を維持する

## 実装方法
1. **記録ファイル**: `kaiwa.md`を使用
2. **記録タイミング**: 
   - セッションの重要な決定や実装が完了した時点
   - 技術的な相談や解決策が得られた時点
   - ユーザーから明示的に記録を求められた時点
3. **記録形式**:
   ```markdown
   ## YYYY-MM-DD - セッションタイトル
   
   ### 実施内容
   - 具体的な作業内容
   
   ### 決定事項・結果
   - 重要な決定や結果
   
   ### 今後のアクション
   - 次のステップ
   ```

## 運用ルール
- 既存のkaiwa.mdファイルに追記する形で記録
- 機密情報や個人情報は記録しない
- 技術的な内容、コード変更、アーキテクチャ決定を重点的に記録
- Gemini CLIとの相談内容も適切に要約して記録
## Gemini CLI 連携ガイド

### 目的
ユーザーが **「Geminiと相談しながら進めて」** （または同義語）と指示した場合、Claude は以降のタスクを **Gemini CLI** と協調しながら進める。
Gemini から得た回答はそのまま提示し、Claude 自身の解説・統合も付け加えることで、両エージェントの知見を融合する。

---

### トリガー
- 正規表現: `/Gemini.*相談しながら/`
- 例:
- 「Geminiと相談しながら進めて」
- 「この件、Geminiと話しつつやりましょう」

---

### 基本フロー
1. **PROMPT 生成**
Claude はユーザーの要件を 1 つのテキストにまとめ、環境変数 `$PROMPT` に格納する。

2. **Gemini CLI 呼び出し**
```bash
PROMPT="ユーザーの要件をここに記載"
gemini <<EOF
$PROMPT
EOF
```

3. **結果の取得と提示**
- Gemini からの回答を完全に取得
- ユーザーに「Geminiからの回答:」として提示
- Claude 自身の分析・補足を追加

---

### 実装例

```bash
# 1. プロンプトの準備
PROMPT="BSFプロジェクトでのMQTT実装について、最適なアーキテクチャを提案してください"

# 2. Gemini CLIの実行
GEMINI_RESPONSE=$(gemini <<EOF
$PROMPT
EOF
)

# 3. 結果の表示
echo "=== Geminiからの回答 ==="
echo "$GEMINI_RESPONSE"
```

---

### 協調作業のパターン

1. **設計相談パターン**
   - ユーザーの要件をGeminiに相談
   - Geminiの提案を基にClaude が実装
   - 実装結果をGeminiでレビュー

2. **問題解決パターン**
   - エラーや課題をGeminiに相談
   - 複数の解決策を取得
   - Claude が最適な解決策を選択・実装

3. **知識統合パターン**
   - 特定の技術についてGeminiに質問
   - Claude の知識と統合
   - より包括的な解決策を提供

---

### 注意事項

- Gemini CLI が利用できない場合は、その旨をユーザーに通知
- Gemini の回答は必ず「Geminiからの回答」として明示
- Claude 自身の見解も併せて提供し、両エージェントの知見を活用
- 機密情報や個人情報はGeminiに送信しない

---
